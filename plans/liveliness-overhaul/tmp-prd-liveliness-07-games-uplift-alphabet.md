## 1. Context

The Alphabet section is 4 games: **Lær Alfabetet** (calm browse, `/alphabet/learn`), **Bogstav Quiz**
(`/alphabet/quiz`), **Hukommelse 10 & 20** (letters memory, `/learning/memory/letters/10|20`). Mechanically they
are solid and the owner is happy with how they *teach*; the gap is purely **look & feel** vs the Structured-World
shell (per PRD-06 §1). Alphabet is the simplest section to re-skin cleanly, so it proves the Foundation language.

**What's flat today (grounded this session):** Bogstav Quiz shows a **flat 2D system emoji** (🐒/🚀/🍎) inside the
frosted `PromptStage` card, with keyboard-key letter tiles. The browse bloom and memory cards likewise show system
emoji. All become the tactile language + baked soft-3D art.

**Content finding (must handle):** there are **two independent, disagreeing** letter→word→emoji manifests:
- `src/config/letterWords.ts` → `LETTER_WORDS` (25 letters; omits Q/W/X/Å) — used by **Bogstav Quiz + Lær Alfabetet**.
- `src/components/learning/MemoryGame.tsx` → inline `LETTER_ICONS` (all 29; different words/emoji) — used by **Memory**.

They diverge (A = Abe 🐒 vs Ananas 🍍; B = Bil 🚗 vs Bjørn 🐻; D = Drage 🐉 vs Due 🕊️; …). See §4. **Decision for the
owner (§6):** reconcile to ONE shared manifest so a letter shows the *same* baked object everywhere (recommended —
consistency is a real quality win for a 5-year-old, and it halves the art), vs keep two sets and bake both.

**Success:** entering Bogstav Quiz feels like the alphabet menu it launched from — the word-picture is a tactile
soft-3D object resting in the calm rainbow/ocean/space/dino world, the letter tiles are pressable clay, the letter
glyphs crisp and central. The letter (the lesson) is as clear as today, or clearer.

## 3. Technical design (per game — consumes the Foundation)

**From the Foundation (assume these exist):** `TactileTile` (soft-3D pressable, `state`/`hint`, glyph-or-img
content), `PromptFocus` (in-world subject on a light-pool + "Hør igen" pill, replaces `PromptStage`), the softened
in-game world veil, the unified HUD (`RepeatButton`/`ScoreChip`), and `src/assets/games/alphabet/` art loading.

### 3.1 Baked-art loader (alphabet)
- Create `src/assets/games/alphabet/index.ts` exporting `alphabetArt: Record<string /*letter*/, string /*webp url*/>`
  (statically bundled imports, like `assets/themes/icons/index.ts`). One image per letter (see §4 / §6 manifest).
- A tiny helper `letterArt(letter): string | undefined` for consumers.

### 3.2 Bogstav Quiz
- **No config/mechanics change** — `generateQuizItem`/`generateOptions`/`round`/`gameId`/hint/audio all stay.
- The **prompt picture** becomes baked art: extend `questionVisual` to allow an art key, OR (simpler) have
  `PromptFocus`/`renderHero` resolve `letterArt(correctLetter)` and render a baked `<img>` (soft-3D, on its
  light-pool) instead of `<HeroEmoji>{emoji}</HeroEmoji>`. Keep the emoji only as a dev fallback (never shipped —
  art-gated). Glyph fallback for letters with no art (shouldn't happen once §4 is complete).
- **Answer tiles**: `AnswerTile` already renders `TactileTile` internally (Foundation F1), so the letter glyphs
  become clay tiles automatically — **glyphs stay type** (the lesson). No change in `AlphabetGame.tsx`.
- Verify the letter stays the obvious focus (subject is the *word-picture*, the answer is the *letter* — unchanged).

### 3.3 Lær Alfabetet
- `LearningGrid` cells become `TactileTile` via the Foundation swap — bare glyph content unchanged.
- The **bloom** (AlphabetLearning's own hero, 176-228): render the baked-art object (`letterArt(letter)`) on a
  light-pool where the emoji is today (212); keep the giant glyph + word. Reuse `PromptFocus` here if it factors
  cleanly, else mirror its light-pool + contact-shadow styling. Q/W/X/Å keep glyph-only bloom.
- Drop the `intro={false}` prop (Foundation F0). Browse-XP + audio unchanged.

### 3.4 Hukommelse (letters)
- Card faces adopt the `TactileTile` material via the Foundation `UnifiedMemoryGame` swap (keep the real-3D flip).
- The card-front **icon** (767-775) renders `letterArt(letter)` baked `<img>` instead of the system emoji; glyph +
  word stay. Requires the memory manifest to resolve to the same art keys — see §6 (consolidation decides whether
  Memory reuses `alphabetArt` or needs its own divergent images).

### 3.5 Per-game UX rework?
Alphabet is mechanically sound — the "rework" here is **material + baked art + manifest consolidation**, not layout
surgery. No mechanic/flow changes proposed. (If play-testing the clay tiles reveals the lip-less tile reads as less
pressable, apply the Foundation's subtle-edge fallback — F1 note.)
