## 0. Learnings carried forward (fold-in from Alphabet `-07` + Math `-08`)

1. **The art loader pattern is verbatim.** A per-section `index.ts` eager-`import.meta.glob('./*.webp', {eager, query:'?url', import:'default'})`, filename-stem → key, exports a sync `xxxArt(id)` returning `string | undefined`. Empty until files land, then auto-registers. Farver copies `src/assets/games/math/index.ts` (the id-keyed variant) to `src/assets/games/farver/index.ts`. See `src/assets/games/alphabet/index.ts` + `math/index.ts` verbatim.
2. **`PromptStage` (the frosted card) is retired per game, not globally.** The Foundation's engine-level swap only reached the **shared engines**. Hand-rolled games + screens that render `PromptStage` **directly** still show the old frosted card and must migrate to `PromptFocus` by hand (`.claude/rules/game-development.md`). In Farver the direct `PromptStage` callers are **Nuancer + Lær Farver** (grep-confirmed §2). `PromptStage.tsx` **stays in-tree** after this area (English/Math/Ordleg still import it until their PRDs) — do **not** delete it.
3. **Consolidate / curate to ONE shared subject set.** Alphabet merged two divergent manifests; math baked one 8-object counting set reused across the section. Farver bakes **one curated object set** (§4, owner-locked §6.1) reused by Farvejagt, Hvilken Farve?, and Lær Farver so a given object looks identical everywhere.
4. **Picture-less / abstract things stay honest.** Numerals stayed glyphs; math's pattern dots stayed emoji (abstract tokens, not depicted subjects). **Farver's analogue is stronger: the colours themselves are the lesson and are DATA, never art.** Colour swatches, mixing droplets, and shade tiles are **pure-hex surfaces** (`COLOR_SWATCH` / `SHADES` / `primaryColors`) — they get the tactile *grounding* re-material (§3.0) but their **fill stays the true hex**; only the *objects* (apple/car/…) are baked.
5. **Screenshot/verify gotcha:** auto-played game TTS doesn't fire headless — identify the showing subject via the baked `<img>` src (or `[data-prompt-focus] img` for PromptFocus screens) and drive feedback with `?fx=correct|wrong|hint`. All four drag games already expose the pure-render `devFx()` harness (§2). Baked into §8.
6. **Frozen-world payoff (F3) is claimed by *removing occluders*.** The alphabet/math boards got calmer by letting the softened world show through. Farvejagt + Ram Farven currently paint an **opaque framed board `<Box>`** (a `3px solid` border + `rgba(...,0.5)` fill) that occludes the world entirely — retiring/softening those is what makes these two boards read "calm-present & grounded" (Decision-log "World presence").

### 3.0 The Farver-specific re-material rule (READ FIRST — it governs every surface below)

Farver has two surface families and they re-material **differently**:

- **Object surfaces** (the apple/car/… draggables + collected-ring items + Hvilken Farve's dragged object + Lær Farver's
  example tiles): become **frameless baked soft-3D objects resting in the world** — a baked `<img>` (`colorObjectArt(id)`)
  with a **`contactShadow(...)` ellipse beneath + `softShadow()`**, **no hex backing square, no border, no lip** (the
  `SceneObject` recipe, mirrored by `TactileTile`'s frameless mode). Fallback = today's emoji while art is un-baked.
- **Colour surfaces** (swatches / droplets / shade tiles / the goal + pot fills): the **fill STAYS the true hex** (it's
  the lesson — `tileSurface`'s white→accent gradient would wash the colour out, so it is **NOT** used here). The
  re-material only swaps the **hard `0 6px 0 darken(hex)` keyboard lip → a grounded `contactShadow(hex)` ellipse +
  `softShadow()`**, so a swatch reads as a clay disc resting in the world instead of a plastic keyboard key. Keep the
  existing thin white ring + the subtle top sheen. `usePointerTilt` on non-touch is optional.

This rule is why Farver can't just "wrap everything in `TactileTile`": colour surfaces need `TactileTile`'s **depth**
(contact shadow + soft shadow + press) but **not** its tinted surface. Implement colour surfaces either as
`TactileTile` with a `sx` background override to the true hex, or by applying `contactShadow`/`softShadow` directly on
the existing pure-hex `Box` (implementer's call; keep it DRY across the 4 games — a tiny local `ColorDisc` helper is fine).

**Reduced-motion:** motion off, but the baked art, contact shadows, light-pools, colour fills, feedback rings, and
audio all still render (Foundation parity).
