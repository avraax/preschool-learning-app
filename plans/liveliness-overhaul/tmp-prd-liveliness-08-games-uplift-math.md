## 0. Alphabet learnings carried forward (fold-in)

Grounded from the shipped alphabet area (`e09f1aa`/`de6b498`/`9a2ba2d`) and its debrief (`696fe52`):

1. **The art loader pattern works verbatim** — a per-section `index.ts` that eager-`import.meta.glob`s `./*.webp`
   (`{ eager, query: '?url', import: 'default' }`), maps each filename stem → key, and exports a sync
   `xxxArt(id)` helper returning `string | undefined`. Empty until files land, then auto-registers. Math copies
   `src/assets/games/alphabet/index.ts` to `src/assets/games/math/index.ts` with number/object-id keys.
2. **`questionVisual.art`→`HeroArt` is the quiz-hero hook.** The default `UnifiedQuizGame` path renders
   `qv.art ? <HeroArt src={qv.art}/> : <HeroEmoji>` (`UnifiedQuizGame.tsx:529-533`); games with a custom
   `renderHero` (Tal Quiz, Hvad Mangler) render `<HeroArt>`/`<HeroEmoji>` themselves. `HeroArt`/`HeroEmoji`
   both live in `PromptStage.tsx` (kept as the home of those helpers; the frosted *card* is what PromptFocus
   retired).
3. **Memory uses a single `iconArt` `<img>`** on the card front (`UnifiedMemoryGame.tsx:769-784`) — the mechanism
   renders exactly one image. Numbers memory wanting a *count* cluster needs a small, backward-compatible engine
   addition (§3.6 / §6), not just data.
4. **Picture-less items stay glyph-only, honestly** (alphabet Q). Math's analogue: **numerals are glyphs and
   never become art**, and abstract pattern tokens (Hvad Mangler's colour dots) are not "depicted subjects" —
   see §3.5 / §6.
5. **Consolidate to ONE shared subject set.** Alphabet merged its two divergent letter→word manifests into one
   so a letter shows the same object everywhere. Math's counting objects must likewise be **one shared set**
   used by Tal Quiz, Lær Tal, Sammenlign, and Memory — so counting "feels" the same across the section.
6. **Screenshot/verify gotcha:** auto-played game TTS usually doesn't fire headless, so identify the showing
   subject via the hero image src (`[data-prompt-focus] img`) + `?seed=<n>`, and drive feedback with
   `?fx=correct|wrong|hint`. (Baked into §8.)

## 1. Context

The Math section is **6 games** off `/math`: **Lær Tal** (calm browse, `/math/numbers`), **Tal Quiz**
(`/math/counting`), **Plus Opgaver** + **Minus Opgaver** (one hand-rolled `MathOperationGame`,
`/math/addition|subtraction`), **Sammenlign Tal** (hand-rolled `ComparisonGame`, `/math/comparison`),
**Hvad Mangler?** (`HvadManglerGame`, `/math/patterns`), and **Hukommelse 10 & 20** (numbers memory,
`/learning/memory/numbers/10|20`). Mechanically they're solid and the owner is happy with how they *teach*
(counting-not-symbol-matching, near-number distractors, the equation-reveal, the krokodille chomp) — the gap
is **look & feel** vs the Structured-World shell (PRD-06 §1).

**What's flat today (grounded this session with `ui-screenshot`, iPad landscape, all games driven):**

**The core finding:** three games (Plus/Minus, Sammenlign, Lær Tal) **never adopted PromptFocus** — they were
hand-rolled / used `PromptStage` directly, so the Foundation's engine-level swap didn't reach them. They are
the concentrated code work. Tal Quiz + Hvad Mangler + Memory already have the material; they need only baked
**counting-object art** where they still show flat emoji.

**Scope (owner-set):** numerals stay glyphs; **baked art = counting objects + memory-card icons** (plus the
Sammenlign krokodille, which is the section's one flat-emoji character). Keep `MathOperationGame`'s
**equation-reveal** and `ComparisonGame`'s **krokodille chomp** exactly.

**Success:** entering any math game feels like the Rummet/Regnbue/Havet/Dino menu it launched from — the
equation/comparison/count rests directly in the calm frozen world on its own light-pool (no frosted glass
panel), the things the child counts are tactile soft-3D objects, and the numerals (the lesson) stay crisp
central type. Mechanics, difficulty ranges, audio, and reward flow are untouched.
