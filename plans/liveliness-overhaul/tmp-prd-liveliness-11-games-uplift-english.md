## 0. Learnings carried forward (fold-in from Alphabet `-07` + Math `-08` + Farver `-09` + Ordleg `-10`)

1. **Both baked-art engine paths already exist — English changes NO shared engine.** PRD-07 added the hero path
   (`questionVisual.art` → `HeroArt`, `UnifiedQuizGame.tsx:535-539`); **PRD-10 added the answer-tile path**
   (`QuizItem.art` → `TileArt`, `UnifiedQuizGame.tsx:716-717`, with `QuizItem.art?: string` at `:72` and
   `questionVisual.art?` at `:66`). English's Word/Translate games need the **hero** path (picture prompt); Lyt og
   Find needs the **answer-tile** path (picture answers). **Both are shipped and backward-compatible.** So unlike
   Ordleg (which had to add the answer-tile path), English touches only its own four component files + a new loader.
2. **The art-loader pattern is verbatim.** A per-section `index.ts` eager-`import.meta.glob('./*.webp', {eager,
   query:'?url', import:'default'})`, filename-stem → key, exports a sync `xxxArt(id): string | undefined`; empty
   until files land, then auto-registers. English copies `src/assets/games/ordleg/index.ts` (the id-keyed variant,
   NOT the alphabet glyph-keyed one) to `src/assets/games/english/index.ts` → `englishArt(id)`.
3. **`PromptStage` (the frosted card) is retired per game/screen, not globally.** The Foundation's engine-level swap
   only reached the shared engines. The three English **quizzes ride `UnifiedQuizGame`** → already on `PromptFocus`
   (no action). But **Lær Engelsk (`EnglishLearning.tsx`) renders `<PromptStage>` DIRECTLY** (`:14`/`:71`) for its
   bloom and so still shows the old frosted card — it must migrate to `PromptFocus` by hand (§3.5), exactly like Sig
   et Ord did in Ordleg. **`PromptStage.tsx` stays in-tree** — `HeroArt`/`HeroEmoji`/`TileArt` still live there and
   `StickerAlbum.tsx:317` still carries the legacy `#ECF1F8` gradient — do **not** delete it.
4. **Consolidate/curate to ONE shared subject set, and only bake what depicts honestly.** English bakes **one shared
   English-word-picture set** (§4) keyed by the English word, reused across all four English surfaces. The discipline
   from every prior area applies to which words get baked:
   - **Numerals stay glyphs / keycaps** (Math §0.4; the "one/two/three…" theme's `1️⃣…🔟` are number tokens, not
     depictable clay objects) → **numbers stay emoji**.
   - **Colours are educational DATA, true-colour, never baked/themed** (Farver rule — `🔴🔵🟢…` are abstract colour
     tokens like Nuancer's ☀️/🌙 chip that stayed emoji) → **colours stay emoji**.
   - **Abstract words with no clean depiction stay emoji** (Ordleg §0.4 — hej/mor/far kept emoji) → the **greetings**
     theme (hello/yes/no/please/…) stays emoji.
   - **People and isolated body parts stay emoji** (Ordleg kept `mor`👩/`far`👨 as emoji; a lone clay eye/tooth/mouth
     reads uncanny, and clay people raise representation issues) → **family + body stay emoji** (owner decision §6).
   - **Concrete, child-safe objects get baked** → **animals + food + objects** (definite), **nature** (recommended).
5. **Screenshot/verify gotcha:** auto-played game TTS doesn't fire headless — identify the showing subject via the
   baked `<img>` src: the prompt via `[data-prompt-focus] img` (Word/Translate), the answer tiles via the per-tile
   `<img>` (Lyt og Find), the browse bloom via `[data-prompt-focus] img` + the card `<img>`s. Drive feedback with the
   engine's `?fx=correct|wrong|hint` harness — **all three English games ride `UnifiedQuizGame`'s `devFx`** (no
   hand-rolled games in this area, so every game has the `?fx=` harness; the browse has none — drive it by tapping).
6. **Frozen-world payoff (F3) is claimed by removing occluders.** English's only hard occluder is **Lær Engelsk's
   frosted `PromptStage` card** over the focal zone; the three quizzes already show the softened world through
   `PromptFocus`. Retiring the browse's `PromptStage` (§3.5) is what makes that one board read "calm-present."
7. **The Lyt og Find answer-reveal guardrail (English-specific, do NOT regress).** Lyt og Find is a **listening**
   game: the prompt is *audio only* (the app speaks the word; the child taps the matching picture). Its hero is the
   neutral **🔊 "wonder card"** (`UnifiedQuizGame.tsx:559-576`, the `quizType==='english'` + no-`questionVisual`
   branch) precisely so the prompt **never reveals the answer**. Baking art for Lyt og Find means baking the
   **ANSWER tiles** (via `QuizItem.art`) — its **prompt stays the 🔊 card; NEVER set `questionVisual.art` on Lyt og
   Find** (that would show the answer). This is the inverse of Word/Translate, where the picture IS the prompt.

### 3.0 The English re-material rule (READ FIRST — governs every surface below)

English has two surface families:
- **Picture surfaces** (Lyt og Find's answer tiles, Word/Translate's prompt picture, Lær Engelsk's bloom + card
  pictures): become **baked soft-3D English-word-pictures** — a `<img src={englishArt(id)}>` via the engine's
  `HeroArt`/`TileArt` (quizzes) or a hand-rendered `HeroArt`/`<img>` (browse). **Fallback = today's emoji** while art
  is un-baked, or for words that are never baked (numbers/colours/greetings/body/family — §0.4/§4).
- **Type surfaces** (the English **word** on Word/Translate answer tiles, the **Danish caption** under the Translate
  prompt): **stay Comic Sans `Typography`** — reading the English word IS the lesson; the Danish caption is a
  scaffold. They ride the `TactileTile`/`PromptFocus` **grounding** (contact shadow + soft shadow + press) but the
  content is always type, **never** baked art.

**Reduced-motion:** motion off, but baked art, contact shadows, light-pools, feedback rings, and audio all still
render (Foundation parity).

### 3.1 No shared-engine change (the key simplification, §0.1)

Unlike Ordleg (which added `QuizItem.art` → `TileArt`), **English adds nothing to `UnifiedQuizGame` or
`PromptStage`.** Both paths are shipped and backward-compatible:
- **Hero picture** (Word/Translate): set `questionVisual.art` → `renderHero` renders `HeroArt` (else `HeroEmoji`
  fallback) and still renders `questionVisual.word` beneath (`UnifiedQuizGame.tsx:535-556`) — so Translate's baked
  picture **+** Danish caption works out of the box (keep `emoji` set as the fallback so the caption uses the small
  `qv.emoji ? …` type size, not the big word-only size — `:547-550`).
- **Answer picture** (Lyt og Find): set `QuizItem.art` → the tile renders `TileArt` (else the `display`
  emoji/`Typography`) (`UnifiedQuizGame.tsx:716-717`). All other quizzes leave both unset → byte-identical.

### 3.2 Baked-art loader (english)

- Create `src/assets/games/english/index.ts` **by copying `src/assets/games/ordleg/index.ts`** (the id-keyed
  variant): eager-`import.meta.glob('./*.webp', {eager, query:'?url', import:'default'})`, filename-stem → key,
  export `englishArtMap: Record<string,string>` + a sync **`englishArt(id): string | undefined`**. Art-gated: empty
  until the batch lands.
- **Id derivation (no word-array edits):** the English word IS the natural key. Add a tiny exported helper
  `englishArtId(en: string): string` = `en.toLowerCase().replace(/[^a-z0-9]/g, '')` (so `"ice cream"` → `icecream`;
  every other concrete word is already a single ASCII token). Consumers call `englishArt(englishArtId(w.en))`.
  Because un-baked / never-baked words (numbers/colours/greetings/body/family) simply have no file, `englishArt`
  returns `undefined` for them → the caller keeps its emoji fallback **automatically**. **No edit to
  `englishVocab.ts` is needed** (no `art` field on `EnglishWord`) — the id is computed from `en`.
- Filenames = the art id (ASCII, e.g. `dog.webp`, `icecream.webp`, `car.webp`). Simpler than Ordleg (no Danish
  glyphs) and simpler than Alphabet (no glyph→letter map).
