# PRD-12 Zero-Emoji — Gemini Art Prompts (paste-ready)

Companion to `tmp-prd-liveliness-12-games-uplift-zero-emoji.md` §4 (Phase B). These are the **only** pictorial
subjects still shown as emoji after the PRD-06…-11 uplift. Generate each in Gemini, then a build session keys them
via `.claude/rules/scene-assets.md` and wires them.

## How to use (read once)
- **Copy ONE prompt at a time** — each is complete on its own (subject + full style guide inlined).
- **Style reference:** on every generation, attach 2–3 existing `src/assets/themes/icons/*.webp` (or the higher-res
  `art-src/icons/*.png`) as STYLE references so the set matches the app. Re-use your first good render as a
  consistency anchor for the rest of that batch.
- **Output:** flat `#00FF00` background, **square, highest resolution, PNG**. Name each file by the **id** shown.
- **Download gotcha:** save via **right-click → "Save image as…"**, NOT the button embedded on the image (that one
  stamps the ✦ sparkle + can composite in stray elements). Verify the render full-size, not the chat thumbnail.
- **Body parts:** depict a **friendly whole cartoon child emphasizing the part** (pointing to / showing it) — NOT a
  disembodied limb (a lone clay limb reads uncanny). This is the whole reason they were emoji before; the
  whole-character framing fixes it.
- Drop the folder path back to the build session when done; it maps files → ids and keys them.
